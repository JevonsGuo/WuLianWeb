# 物联网设备公司产品展示网站 - 完整技术需求文档

## 一、项目概述

### 1.1 项目目标
构建一个基于 **Next.js 14 (App Router) + TypeScript + Supabase + Vercel** 的公司产品展示网站，包含：
- 公司首页
- 产品展示页（核心交互：左侧大类 + 右侧展示柜动画 + 产品弹窗）
- 解决方案页（按行业展示）
- 简易后台管理（产品/大类/解决方案的增删改查）

### 1.2 技术栈
| 类别 | 技术 |
|------|------|
| 框架 | Next.js 14 (App Router) |
| 语言 | TypeScript |
| 数据库 & 存储 | Supabase (PostgreSQL + Storage) |
| 样式 | TailwindCSS |
| 弹窗组件 | Radix UI Dialog |
| 部署 | Vercel |
| 动画 | CSS Transition + 微交互 |

### 1.3 数据录入方式
- 提供后台管理界面（`/admin`），支持：
  - 产品大类的增删改查
  - 产品的增删改查（关联大类）
  - 解决方案的增删改查
- 内置 AI 生成的示例数据（通过迁移脚本或后台初始化按钮）

## 二、页面详细设计

### 2.1 公司首页 (`/`)

#### 布局模块
1. **Hero 区域**
   - 公司 Slogan（大标题）
   - 简短介绍文字
   - 背景图片（静态，可配置）

2. **产品大类快速入口**
   - 展示前 3 个大类（图片 + 名称）
   - 点击跳转到 `/products` 并自动选中对应大类

3. **解决方案行业快速入口**
   - 展示前 3 个行业（图片 + 行业名称）
   - 点击跳转到 `/solutions`

4. **页脚**
   - 公司名称
   - 邮箱
   - 地址
   - 备案号（如有）

> 首页内容部分静态配置（硬编码），无需后台管理。

---

### 2.2 产品展示页 (`/products`)

#### 布局结构（左右两栏，桌面优先）

**左侧边栏（宽度约 280px）**
- 垂直排列所有产品大类
- 每个大类显示：
  - 缩略图（40x40px，圆角）
  - 大类名称
- 选中状态：背景色高亮（如 `bg-blue-50`），左侧加边框指示条
- 点击切换大类时，右侧触发动画

**右侧内容区（自适应宽度）**
- 展示当前选中大类下的所有产品
- 产品以**网格布局**展示（每行 3~4 个卡片）
- 每个产品卡片：
  - 产品图片（正方形，`object-cover`，hover 轻微放大）
  - 产品名称（粗体，居中或左对齐）
  - 产品型号（较小字号，灰色）
- 点击**图片**打开弹窗（不是整个卡片）

#### 动画效果（切换大类时）
```css
/* 推荐动画：淡入 + 轻微上移/缩放 */
.product-grid {
  transition: opacity 0.2s ease, transform 0.2s ease;
}
.product-grid.fade-out {
  opacity: 0;
  transform: scale(0.98);
}
.product-grid.fade-in {
  opacity: 1;
  transform: scale(1);
}

切换时：右侧网格内容先淡出 → 更新数据 → 淡入
可选：每个产品卡片依次出现（stagger 效果，延迟 0.03s * index）
产品弹窗（使用 Radix UI Dialog）

弹窗内容：

产品名称（标题）
产品型号（副标题）
产品介绍（支持换行/段落）
产品图片（弹窗内可显示更大版本，复用格子里的图片）
底部两个按钮：

2.3 解决方案页 (/solutions)

布局

页面上方：标题 + 简短介绍
下方：垂直排列行业卡片（每个行业一个区块）

左侧：行业图片（如生产线场景）
右侧：

行业名称（如“汽车制造业”）
解决方案描述（段落文字）
关联产品推荐：列出该行业常用的产品（按钮或标签，点击跳转到 /products 并定位到对应产品？或直接链接到产品弹窗？简化版：链接到产品页并自动滚动到该产品）
简化实现：关联产品只显示产品名称和型号，点击后在新标签页打开 /products?productId=xxx 并自动打开弹窗（需要 URL 参数支持）。第一版可以不实现自动弹窗，仅跳转到产品页让用户手动点击。
三、数据库设计（Supabase）

表结构

3.1 product_categories

字段	类型	说明
id	uuid (PK)	默认 gen_random_uuid()
name	text (not null)	大类名称
image_url	text	左侧列表显示的图片 URL（Supabase Storage 或外部）
sort_order	integer (default 0)	排序，数字越小越靠前
created_at	timestamptz	自动
updated_at	timestamptz	自动
3.2 products

字段	类型	说明
id	uuid (PK)	
category_id	uuid (FK → product_categories.id)	关联大类
name	text (not null)	产品名称
model	text (not null)	产品型号
description	text	详细介绍（纯文本，支持换行）
image_url	text	产品卡片图片 URL
manual_url	text	产品手册下载链接（PDF）
certificate_url	text	证书下载链接（PDF）
sort_order	integer (default 0)	在大类内的排序
created_at	timestamptz	
updated_at	timestamptz	
3.3 solutions

字段	类型	说明
id	uuid (PK)	
industry_name	text (not null)	行业名称（如“汽车制造”）
image_url	text	行业配图
description	text	解决方案描述
related_product_ids	text	存储产品 ID 数组，如 ["id1","id2"]（JSON 字符串）
sort_order	integer	排序
created_at	timestamptz	
注意：related_product_ids 使用 Supabase 的 text[] 类型更合适，但为了简单，可以用 JSON 字符串，前端解析。
